ПОЛИЗ обладает следующими свойствами:
— Любой элемент ПОЛИЗа — либо инструкция, либо операнд одного из четырёх доступных типов. Адреса инструкций (используемые при переходах) и номера переменных хранятся в виде операндов типа Integer.
— Инструкции хранят в себе информацию о типах операндов (например, «+» для вещественных чисел и «+» для строк — разные операции); проверка на применимость операций к конкретным операндам не производится (это возложено на семантический анализ); при необходимости осуществляется неявное приведение типов. Для явного приведения типов может быть использован унарный плюс соответствующего типа.
— Программа не хранит никакой информации о названиях и типах переменных; единственное хранимое значение — их количество. Обращение к ним идёт по номерам, при этом в любую переменную можно записать любое значение; единственное ошибочное действие — попытка прочесть неинициализированную переменную.

Кроме арифметических операций (семантика и обозначения которых очевидны), ПОЛИЗ может содержать следующие операции:
— Очистка кэша (при дампе обозначается символом «;»).
— Переход по лжи (обозначается «F»). Достаёт из стека адрес инструкции, затем — значение типа Boolean; если значение ложно, осуществляет переход на указанный адрес.
— Загрузки переменной («l»). Достаёт из стека номер переменной и кладёт в стек её значение.
— Сохранения переменной («s»). Достаёт из стека номер переменной и записывает в указанную переменную верхушку стека (не удаляя!).
— Записи («w»). Достаёт из стека значение и выводит его на экран.
— Перевода строки («W»). Выводит на экран перевод строки.
— Чтения («r»). Читает строку из консоли и кладёт её в стек в виде строкового значения. Предполагается, что после этого она будет приведена к нужному типу и записана в переменную.
— Дублирования верхушки стека («d»).
Операции «больше или равно» и «меньше или равно» при дампе ПОЛИЗа обозначаются круглыми скобками «(» и «)» соответственно.
